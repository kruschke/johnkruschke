> Then on the serial processing tables I'm a little confused.  When
> the target is absent, the start time is 400msec, percieve an item is
> 20msec, target present is30msec, and another item is also 20msec.
> So the subtotal would be 70 msec and then do I multiply that by 4???

When the target is absent and there are 4 items, yes, that's right, because in serial search you need to search successively through all 4 items before you know that the target is not there. 

> and when the target is presen do do I multiply by 2.5???

Right, because on average you'd search 2.5 items before finding the target.
